Backtracking

General

Backtracking is an algorithm for finding all possible solutions to some computational problems. The solutions are found by incrementally building candidates for possible solutions and checking The solutions are found by incrementally building candidates for possible solutions and checking for its validity. If it determines that the candidate cannot be part of a valid solution, it abandons that configuration. Backtracking can be applied to problems that allows for partial solutions, meaning part of a configuration can be used to determine if it is a member the full solution. Backtracking allows the possibility of eliminating large number of solutions in a single test; this makes backtracking, often much faster than a brute force approach.

Backtracking for 8 Queens

Place the queens on one column at a time starting in the first column. Every time a queen is placed in a square, there are eight possible places to place a queen in the next column. If a queen cannot be placed in a column safely and it runs out of rows to check, the program must backtrack to the previous column and continue to the next row. If a queen is safely placed in the last column, it means that is one possible solution to the 8 Queens problem.

Pointers

Pointers

A pointer stores the address of another variable in memory
A pointer is declared with an *
int* x = &p; Declares a pointer called x and it is storing the address of p
*x is a reference to the variable p
we can reassign values to p by using *x
We can deference a pointer by using the *also
Dereferencing a pointer gives the value that pointer was pointing to

int x = 5; int* p = &x; // declare an int pointer and assign x's address to it cout << p << endl; // print the address that p is pointing to cout << *p << endl; // print the value that p is pointing to *p = 6; // assign 6 to x cout << x << endl; // print x (6) int& r = x; // declare r as a reference to x r = 7; // assign 7 to x cout << r; // print x (7) return 0;

Arrays as Pointers

An array is a fixed sized collections of elements of the same type, stored continuously in memory.
Array names are not pointers. However they are converted to pointers when we do arithmetic
When you +1 to a pointer, you increase the address by the size of what it is pointing to.
When you add to an array name, you point to an element in that array.
In a 2D array, each element is a row in the array
Sizeof(arrayName) gives you the size of the array
Sizeof(pointer) gives the size of the pointer

// b is a 2D array, so each element is a 1D array. //assuming int are 4bytes and pointers are 8 byte int b[3][4]; cout << sizeof(b) << endl; // size of 2D array = 3*4*4 cout << sizeof(b+0) << endl; // size of pointer to 1st 1d array = 8 cout << sizeof(*(b+0)) << endl; // size of 1st 1d array = 4*4 cout << "The address of b is: " << b << endl; // address of b cout << "The address of b+1 is: " << b+1 << endl; // address of b + (4*4) // &b is a pointer to the entire array, rather than the 1st element. // It's the same address but a different type, so adding 1 doesn't have the same effect. cout << "The address of &b is: " << &b << endl; // address of b cout << "The address of &b+1 is: " << &b+1; // address of b + (3*4*4) return 0;
